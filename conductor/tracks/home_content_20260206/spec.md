# Track Specification: 메인 페이지 히어로 및 기술력 섹션 업데이트 (home_content)

## 1. Overview
메인 페이지의 시각적 요소와 핵심 기술력 섹션을 업데이트하여 회사의 전문성과 실적을 더욱 직관적이고 매력적으로 전달합니다.

## 2. Functional Requirements

### 2.1 히어로 섹션 이미지 교체
- **내용:** 기존 메인 히어로 이미지를 신규 자산으로 교체 및 슬라이드 구성.
- **자산:** 
  - `assets/img/메인_히어로/신구대식물원_방제.jpg`
  - `assets/img/메인_히어로/정부청사_소나무전정.jpg`
- **기능:** 이미지가 부드럽게 전환되는 페이드 또는 슬라이드 효과 적용.

### 2.2 '결과로 증명하는 기술력' 섹션 개편
- **분류 수정:** 기존 3개 카테고리에서 아래 7개 카테고리로 세분화.
  1. 녹지관리
  2. 소나무 전정
  3. 계절꽃 식재
  4. 조경식재
  5. 병충해 방제
  6. 수목수세회복
  7. 위험목 제거
- **레이아웃 (섹션별 그리드):**
  - 각 카테고리별로 독립된 카드/컨테이너 배치.
  - 각 컨테이너 내부에 해당 카테고리의 모든 사진(2~5장)을 그리드 형태로 노출.
  - **이미지 태그 표시:** 사진 이름(before, after, 작업 등)을 파악하여 이미지 위에 배지(Badge) 형태의 태그 노출.
- **디자인 전략:**
  - 가로형/세로형 사진이 섞여 있는 점을 고려하여 Masonry 또는 유연한 그리드 레이아웃 적용.
  - 사진의 의미(Before/After 비교 등)가 직관적으로 전달되도록 태그 시인성 확보.

## 3. Non-Functional Requirements
- **반응형 대응:** 모바일 환경에서도 7개 섹션이 보기 좋게 나열되고 이미지가 깨지지 않아야 함.
- **성능:** 다량의 고화질 이미지 로딩 속도를 고려하여 이미지 최적화 및 Lazy Loading 적용.

## 4. Acceptance Criteria
1. 히어로 섹션에 지정된 2장의 이미지가 정상적으로 노출되고 전환되는가?
2. 기술력 섹션이 7개 분류로 정확히 나뉘어 있는가?
3. 각 분류 내의 사진들이 태그와 함께 누락 없이 노출되는가?
4. 가로/세로 비율이 다른 사진들이 깨지지 않고 레이아웃 내에 조화롭게 배치되었는가?

## 5. Out of Scope
- 각 기술력 항목 클릭 시 이동하는 별도의 상세 페이지 구현 (이번 트랙에서는 메인 페이지 노출에 집중).
