# Track Specification: 주요 사업 페이지 고도화 및 사례 연동

## 1. Overview
기존의 단순한 사업 소개 페이지를 한 권의 브랜드 브로슈어를 읽는 듯한 **스토리텔링형 레이아웃**으로 전면 개편합니다. 각 사업 영역의 전문성을 시각적으로 강조하고, 실제 백엔드에 등록된 **시공 사례와 자동으로 연동**하여 신뢰도를 극대화하는 것이 목적입니다.

## 2. Functional Requirements
### 2.1 브로슈어 스타일 섹션 구성
- **섹션 1: 나무병원 사업**
  - 핵심 가치: "정확한 진단과 과학적 치료"
  - 내용: 수목 진단·처방, 치료·방제, 생육환경 개선, 위험수목 진단 등.
  - 디자인: 대형 시공 사진과 구조화된 타이포그래피를 활용한 시원한 레이아웃.
- **섹션 2: 조경식재 사업**
  - 핵심 가치: "사람과 자연이 숨 쉬는 공간 창조"
  - 내용: 조경 설계·시공, 유지관리, 대형목 이식, 도시재생 녹지 개선 등.
  - 디자인: 시각적 균형을 맞춘 전문 조경 포트폴리오 스타일.

### 2.2 시공 사례 자동 연동 (태그 기반)
- 각 사업 섹션 하단에 **[관련 사례 보기]** 영역 추가.
- 백엔드 API를 통해 해당 카테고리(`나무병원` 또는 `조경식재`)로 분류된 최신 시공 사례 3~4개를 자동으로 리스트업.
- 사례 클릭 시 해당 시공 사례 상세 페이지로 이동.

### 2.3 백엔드 카테고리 필드 확장
- 시공 사례(`Performance`) 모델에 `category` 필드 추가 (또는 기존 태그 로직 활용).
- 관리자 등록 폼에서 '나무병원' / '조경식재'를 선택할 수 있는 기능 제공.

## 3. Non-Functional Requirements
- **디자인 일관성:** '회사소개' 페이지에서 정립된 전문적이고 중후한 디자인 시스템(max-w-7xl, Black 폰트 등) 유지.
- **인터랙티브 UX:** 스크롤 시 각 사업 섹션의 요소들이 부드럽게 나타나는 `framer-motion` 애니메이션 적용.
- **반응형 최적화:** 모바일에서도 브로슈어의 가독성이 유지되도록 이미지와 텍스트 배치 최적화.

## 4. Acceptance Criteria
- [ ] 각 사업 섹션이 브로슈어 스타일의 시원한 레이아웃으로 구현되었는가?
- [ ] 시공 사례 등록 시 선택한 카테고리에 맞춰 해당 섹션 하단에 자동으로 노출되는가?
- [ ] '회사소개' 페이지와 디자인 톤앤매너가 일치하는가?
- [ ] 스크롤 애니메이션이 과하지 않고 전문적인 느낌을 주는가?

## 5. Out of Scope
- 관리자 페이지에서의 사업 내용 텍스트 직접 수정 (현재는 코드/JSON 기반 관리).
- 각 사업 분야별 별도의 독립 페이지 생성 (단일 페이지 내 섹션 구성).
