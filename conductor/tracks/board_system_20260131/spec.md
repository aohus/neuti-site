# Track Specification: 게시판 시스템 (공지사항 및 시공/견적문의) 구현

## Overview
'느티나무 협동조합 나무병원' 홈페이지 내에서 소식을 알리는 '공지사항'과 고객의 문의를 처리하는 '시공/견적문의' 게시판을 구축합니다. 이를 위해 **단일 admin 계정 기반의 간단한 인증 시스템**을 포함하여 권한 관리를 수행합니다.

## Functional Requirements
- **인증 시스템 (Authentication):**
    - 단일 관리자 계정(ID/PW)을 통한 로그인/로그아웃 기능.
    - JWT 또는 세션을 이용한 로그인 상태 유지.
- **공통 기능:**
    - 게시글 목록 조회 (번호, 제목, 작성자, 작성일, 조회수 포함).
    - 게시글 상세 조회.
    - 페이징 (하단 페이지 번호 표시) 및 키워드 검색 (제목/내용) 기능.
    - 전통적인 표(Table) 형식의 레이아웃 적용.
- **공지사항 (Notice):**
    - **로그인한 관리자**만 작성/수정/삭제 가능.
    - 일반 사용자는 읽기만 가능.
- **시공/견적문의 (Q&A):**
    - 누구나 작성 가능.
    - **비밀글 기능:** 작성 시 비밀번호를 설정하여, 작성자와 관리자만 게시글을 조회할 수 있도록 보호.
    - **답변 기능:** **로그인한 관리자**만 게시글에 답변 작성 가능. 목록에 '답변완료' 상태 표시.

## Non-Functional Requirements
- **보안:** 관리자 및 게시글 비밀번호의 안전한 해싱 저장.
- **사용자 경험(UX):** 로그인 상태에 따라 작성/수정 버튼의 노출 여부 제어.
- **디자인:** `product-guidelines.md`에 따른 정갈하고 신뢰감 있는 디자인 유지.

## Acceptance Criteria
- [ ] `/login` 페이지를 통해 관리자 로그인이 가능한가?
- [ ] 관리자 로그인 상태에서만 공지사항 작성 버튼이 보이는가?
- [ ] 시공/견적문의 작성 시 설정한 비밀번호로 글 조회가 제한되는가?
- [ ] 관리자가 문의글에 답변을 작성하면 목록에 '답변완료' 표시가 뜨는가?
- [ ] 검색 및 페이징 기능이 정상 작동하는가?

## Out of Scope
- 일반 사용자 회원가입 기능.
- 복잡한 권한 그룹 관리 (Admin 외 등급 없음).
- 게시글 내 풍부한 텍스트 에디터(WYSIWYG).
