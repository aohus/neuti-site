# Track Specification: 시공 사례(구 주요 실적) 페이지 고도화 및 관리 기능 구현

## 1. Overview
현재 정적인 '주요 실적' 페이지를 관리자가 직접 콘텐츠를 관리할 수 있는 동적 게시판 형태로 전환합니다. 블로그 스타일의 상세 페이지를 통해 시공 과정과 결과를 상세히 공유함으로써 고객의 신뢰도를 높이고 수주 경쟁력을 강화하는 것이 목적입니다.

## 2. Functional Requirements
### 2.1 게시글 목록 (사용자/관리자 공통)
- 기존 레이아웃(갤러리/리스트 형태) 유지.
- 각 항목 클릭 시 해당 시공 사례의 상세 페이지로 이동.
- 페이징 또는 무한 스크롤을 통한 데이터 로딩.

### 2.2 게시글 상세 페이지 (사용자/관리자 공통)
- 블로그 포스트 형태의 레이아웃.
- 상단: 제목, 시공 날짜, 발주처 등 기본 정보 요약.
- 본문: 관리자가 작성한 이미지와 텍스트가 순차적으로 배치되는 자유로운 형식.

### 2.3 관리자 기능 (관리자 전용)
- **접근 권한:** JWT 기반 관리자 로그인 시에만 접근 가능.
- **게시글 작성:**
  - 제목 입력.
  - 대표 이미지(Thumbnail) 설정.
  - 본문 편집 (Rich Text Editor 또는 이미지/텍스트 블록 추가 방식).
  - 이미지 다중 업로드 지원.
- **게시글 수정/삭제:** 기존 게시글의 내용을 변경하거나 삭제하는 기능.

## 3. Non-Functional Requirements
- **이미지 최적화:** 업로드 시 자동으로 리사이징 또는 압축하여 로딩 성능 확보 (WebP 권장).
- **반응형 디자인:** 모바일에서도 블로그 글이 읽기 편하도록 가독성 중심의 레이아웃 적용.
- **보안:** 관리자 API 호출 시 유효한 토큰 검증 필수.

## 4. Acceptance Criteria
- [ ] 관리자로 로그인 시 '글쓰기' 버튼이 노출되는가?
- [ ] 이미지와 텍스트가 포함된 게시글이 정상적으로 저장되고 목록에 나타나는가?
- [ ] 목록에서 항목 클릭 시 상세 페이지로 이동하며, 작성한 내용이 그대로 표시되는가?
- [ ] 로그인하지 않은 일반 사용자는 글쓰기/수정/삭제 기능을 사용할 수 없는가?

## 5. Out of Scope
- 게시글에 대한 사용자 댓글 기능.
- SNS 공유 버튼 (추후 트랙에서 검토 가능).
- 게시글 카테고리 분류 (이번 단계에서는 단일 리스트로 구성).
