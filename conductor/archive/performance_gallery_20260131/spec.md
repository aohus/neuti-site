# Track Specification: 주요 실적 페이지 갤러리 구현

## Overview

'느티나무병원 협동조합'의 전문성과 신뢰도를 시각적으로 전달하기 위해, 과거 공사 실적을 갤러리 형태로 전시하는 페이지를 구현합니다. 사용자는 카테고리 필터와 검색을 통해 원하는 실적을 쉽게 찾을 수 있어야 합니다.

## Functional Requirements

- **데이터 관리 (Content Management):**
  - 실적 데이터는 코드와 분리하여 `frontend/src/data/projects.json` 파일로 독립적으로 관리합니다.
- **실적 리스트 전시:**
  - 분리된 데이터 파일을 기반으로 실적 목록을 표시합니다.
  - 각 항목은 카드 UI로 구성되며 이미지, 제목, 요약문, 태그를 포함합니다.
- **반응형 그리드 레이아웃:**
  - 데스크톱: 3~4열 그리드.
  - 태블릿: 2열 그리드.
  - 모바일: 1열 리스트.
- **필터링 및 검색:**
  - 카테고리(예: 아파트, 기업, 수목치료 등)별 필터 탭 제공.
  - 공사명 및 내용을 대상으로 하는 키워드 검색 기능 제공.
- **페이징:**
  - 하단에 페이지 번호를 이용한 표준 페이징 처리.

## Non-Functional Requirements

- **유지보수성:** 콘텐츠 데이터 구조가 직관적이고 확장이 용이해야 합니다.
- **시각적 완성도:** `product-guidelines.md`에 정의된 자연 친화적(Green/Brown) 테마를 유지합니다.
- **성능:** 이미지 최적화(Next.js Image 컴포넌트 사용)를 통해 빠른 로딩 속도를 유지합니다.

## Acceptance Criteria

- [ ] 실적 데이터 파일(`projects.json`)을 수정했을 때 화면에 정상적으로 반영되는가?
- [ ] 카테고리 탭 클릭 시 해당 항목들만 필터링되어 표시되는가?
- [ ] 검색어 입력 시 관련 실적이 검색 결과에 노출되는가?
- [ ] 페이징 클릭 시 해당 페이지의 데이터로 정상 교체되는가?
- [ ] 모바일 기기에서 1열로 레이아웃이 정상적으로 변경되는가?

## Out of Scope

- 실적 상세 페이지 구현.
- 관리자 페이지에서의 실적 CRUD 기능 (파일 기반 관리로 대체).
