# Track Specification: 수목진단의뢰 페이지 구현

## Overview
고객이 수목의 건강 상태를 전문가에게 의뢰할 수 있는 전용 페이지를 구축합니다. 기존의 HWP 양식 다운로드 방식과 현대적인 온라인 폼 제출 방식을 병행하여 고객 편의성을 높입니다.

## Functional Requirements
- **의뢰서 파일 제공:**
    - `./assets/의뢰서/수목진단의뢰.hwp` 파일을 다운로드할 수 있는 명확한 링크/버튼 제공.
- **온라인 의뢰 폼:**
    - 의뢰자 정보 (성함/업체명, 연락처, 이메일).
    - 대상 수목 정보 (주소, 수량, 주요 증상 설명).
    - 사진 업로드 (현장 수목 사진 등).
- **데이터 처리:**
    - 제출 시 데이터를 데이터베이스(PostgreSQL)에 저장.
    - 제출 완료 후 관리자에게 알림 이메일 발송.
- **레퍼런스 반영:**
    - `./assets/page_reference/진단의뢰.png`의 레이아웃 및 디자인 요소 참고.

## Non-Functional Requirements
- **사용자 경험(UX):** 폼 제출 완료 후 성공 메시지 및 향후 절차 안내문 표시.
- **보안:** 폼 데이터 유효성 검사 및 파일 업로드 시 보안(확장자 제한 등) 고려.
- **디자인:** `product-guidelines.md`에 따른 자연 친화적인 테마 유지.

## Acceptance Criteria
- [ ] 수목진단의뢰.hwp 파일이 정상적으로 다운로드되는가?
- [ ] 온라인 폼의 모든 필수 항목을 입력하지 않으면 제출이 차단되는가?
- [ ] 파일 업로드 기능이 정상 작동하고, 서버/DB에 저장되는가?
- [ ] 폼 제출 시 관리자 이메일로 알림이 오고, DB에 기록이 남는가?
- [ ] 모바일 환경에서도 폼 입력이 편리하게 구성되었는가?

## Out of Scope
- 관리자가 DB에 저장된 의뢰 목록을 조회/관리하는 UI (추후 관리자 기능 트랙에서 구현).
- 이메일 답장 자동화 기능.
